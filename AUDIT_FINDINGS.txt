8. WALLET DOUBLE-DEDUCTION BUG
   Location: dragonvstiger.js, line 136
   ───────────────────────────────────────────────────────
   PROBLEM: Wallet deducted in code AND in database separately
   
   Current Code (line 133-140):
   const updatedWallet = specificUserSocket.wallet - betAmount;
   specificUserSocket.wallet = updatedWallet;
   
   con.query("UPDATE users SET wallet_balance = wallet_balance - ? WHERE id = ?", 
     [updatedWallet, user_Id], ...);
   
   ISSUE: If betAmount = 100, code does: wallet -= 100
          Then database UPDATE subtracts the RESULT, not the amount
          
   Race condition: If two concurrent bets happen:
   - User starts with 1000
   - Bet 1: 500 (wallet becomes 500)
   - Bet 2: 300 (wallet becomes 200)
   - But DB might execute: 1000-500=500, then 500-200=300
   - Result: Wrong balance
   
   FIX: Use single atomic database operation:
   con.query(
     "UPDATE users SET wallet_balance = wallet_balance - ? WHERE id = ?", 
     [betAmount, userId], callback);
   
   Do NOT deduct in code first. Let database be the source of truth.
